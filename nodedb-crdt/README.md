# `nodedb-crdt`

> CRDT engine using Loro, with declarative conflict policies

Loro-based CRDT integration for offline-first sync. Supports per-collection conflict-resolution policies, dead-letter queue for constraint violations, and Multi-Raft commit-phase invariant checks (UNIQUE, FK).

## Status

Pre-1.0. APIs may change between minor versions until 1.0. See the
workspace [`README.md`](../README.md) for the full project overview and
the GitHub [release notes](https://github.com/NodeDB-Lab/nodedb/releases)
for per-version changes.
